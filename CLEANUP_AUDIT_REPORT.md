## Phase 4 — Risk Notes Before Cleanup

1. **Do not touch migrations.** All migration files are part of the database schema history. Removing them will break `python manage.py migrate` on a fresh database.

2. **Do not delete `apps/reports/services.py`** — It is actively imported by `reports/admin.py` and drives the Income Statement, General Ledger, and Sales Report in the admin.

3. **Do not delete `apps/reports/models.py`** — It contains the unmanaged proxy models (`SalesReport`, `GeneralLedger`, `IncomeStatement`) that make admin sections appear in the sidebar.

4. **Do not delete `apps/reports/admin.py`** — It is the live admin for all three reports with full Excel export logic.

5. **The `core/` directory** is NOT in `INSTALLED_APPS` but is directly imported via `from core.admin_site import custom_admin_site` in every admin file. Do not try to "add it to INSTALLED_APPS" without testing first — it will alter admin URL resolution behavior.

6. **`apps/customers/views.py::import_vehicles_from_excel`** is imported in `apps/vehicles/urls.py`. This cross-app function reference is intentional. Do not delete it.

7. **Signals in `apps/rentals/models.py`** are defined directly in models.py using `@receiver` (not in a separate signals.py). They are active without any `AppConfig.ready()` import. Correct pattern for this project.

8. **`apps/accounting/resources.py`** is imported in `accounting/admin.py` (`from .resources import AccountResource`). Keep it.
